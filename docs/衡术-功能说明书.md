# 衡术 Hengshu 功能说明书

> 面向第一次了解衡术的用户，说明各个模块能做什么、在哪里使用、适合什么场景。

衡术是一个 AI Skill 平台。用户可以在这里发现、试用、安装、运行和管理 AI Skill，也可以查看同一个 Skill 在不同模型上的真实运行效果。

---

## 1. 首页

### 这个模块是做什么的

首页是衡术的入口页，用来快速了解平台定位，并进入主要功能区。

### 用户可以做什么

- 了解衡术是什么
- 进入 Skill 市场
- 查看推荐 Skill
- 进入排行榜、模型榜、失败知识库、悬赏区
- 登录、注册或进入个人控制台

### 适合什么场景

新用户第一次访问平台时，可以先从首页了解平台提供哪些能力。

---

## 2. Skill 市场

### 这个模块是做什么的

Skill 市场是浏览和发现 AI Skill 的地方。每个 Skill 都对应一种可复用的 AI 能力，例如标题生成、文章总结、内容改写、结构化输出等。

### 用户可以做什么

- 浏览全部已发布 Skill
- 按分类筛选 Skill
- 按最新、热门、评分、运行表现等维度排序
- 搜索特定 Skill
- 进入 Skill 详情页查看完整说明

### 适合什么场景

当用户有一个具体任务，但不想从零写 Prompt 时，可以先在市场里找现成 Skill。

---

## 3. Skill 详情页

### 这个模块是做什么的

Skill 详情页用于帮助用户判断一个 Skill 是否值得使用。

### 用户可以做什么

- 查看 Skill 名称、介绍、作者、分类和版本
- 查看需要填写的输入字段
- 查看输出格式说明
- 查看 SkillRank、成功率、平均成本、平均耗时、调用量等指标
- 在线试用 Skill
- 下载 YAML / JSON 格式的 Skill Manifest
- 收藏 Skill
- Fork Skill
- 查看不同模型上的兼容表现
- 查看评论、评分和失败反馈

### 适合什么场景

用户想确认“这个 Skill 能不能解决我的问题”“应该用哪个模型跑”“别人用得怎么样”时，可以重点看详情页。

---

## 4. 在线试用

### 这个模块是做什么的

在线试用让用户不用安装本地 Runner，也能直接在网页中运行 Skill。

### 用户可以做什么

- 根据输入字段填写任务参数
- 选择运行模式或模型
- 查看输出结果
- 查看本次运行的耗时、成本、格式是否稳定
- 判断是否要收藏、下载或长期使用这个 Skill

### 适合什么场景

用户第一次看到某个 Skill，想先试一下效果，再决定是否安装或收藏。

---

## 5. 多模型对比

### 这个模块是做什么的

多模型对比用于比较同一个 Skill 在不同模型上的运行表现。

### 用户可以做什么

- 同时选择多个模型运行同一个输入
- 对比不同模型的输出结果
- 对比成功率、格式稳定性、运行耗时和成本
- 判断哪个模型更适合当前 Skill

### 适合什么场景

用户不确定该用便宜模型还是高质量模型时，可以通过对比结果做选择。

---

## 6. 模型中立榜

### 这个模块是做什么的

模型中立榜基于真实运行数据展示模型表现，重点看成功率、格式稳定性、延迟、价格和性价比。

### 用户可以做什么

- 查看不同模型的综合表现
- 按性价比、质量、速度、价格排序
- 判断某个模型是否稳定
- 查看样本数量是否足够
- 识别“BYOK 更省”的模型

### 适合什么场景

用户准备选择长期使用的模型，或者想比较多个模型的实际表现时，可以查看模型中立榜。

---

## 7. 排行榜

### 这个模块是做什么的

排行榜展示平台内表现较好的 Skill 和贡献用户，帮助用户快速发现高质量内容。

### 用户可以做什么

- 查看高分 Skill
- 发现近期表现好的 Skill
- 查看贡献较高的用户
- 通过排名快速进入详情页

### 适合什么场景

用户没有明确关键词，只想看看平台上哪些 Skill 更值得试用时，可以先看排行榜。

---

## 8. 失败知识库

### 这个模块是做什么的

失败知识库聚合 Skill 或模型运行中的常见失败模式，帮助用户少踩坑。

### 用户可以做什么

- 查看常见错误类型
- 查看错误症状和可能原因
- 了解公开修复方向
- 登录后查看修复模板和复验步骤
- 找到某个模型或 Skill 为什么容易失败

### 适合什么场景

当用户遇到 JSON 输出失败、模型截断、格式不稳定、运行报错等问题时，可以先查失败知识库。

---

## 9. 本地 Runner

### 这个模块是做什么的

Runner 是衡术的本地命令行工具，用来登录账号、安装 Skill、在本地运行 Skill，并选择是否回传兼容数据。

### 用户可以做什么

- 登录衡术账号
- 安装 Skill 到本地
- 查看已安装 Skill
- 在本地运行 Skill
- 使用自己的模型接口和 API Key
- 回传兼容报告
- 检查本地环境是否可用
- 更新本地 Skill

### 适合什么场景

适合有自己模型 API、希望本地管理 Skill、或愿意贡献兼容数据的用户。

---

## 10. 设备授权

### 这个模块是做什么的

设备授权用于让命令行 Runner 安全登录衡术账号。

### 用户可以做什么

- 在终端获取设备码
- 在网页中确认授权
- 将本地 Runner 绑定到自己的账号
- 后续在控制台查看或撤销 Runner 实例

### 适合什么场景

用户第一次使用本地 Runner 时，需要通过设备授权完成登录。

---

## 11. 我的控制台

### 这个模块是做什么的

控制台是用户自己的工作台，用于管理账号、Skill、运行记录、通知、术值和 credit。

### 用户可以做什么

- 查看个人概览
- 查看通知
- 查看我的作品
- 发布 Skill
- 查看已安装 Skill
- 管理 Runner 实例
- 查看运行记录
- 查看术值流水
- 使用术值兑换 credit
- 查看收藏
- 查看邀请码
- 修改个人设置

### 适合什么场景

用户登录后，需要管理自己的 Skill、模型 Key、运行历史和平台资产时，都可以进入控制台。

---

## 12. 运行记录

### 这个模块是做什么的

运行记录保存用户每次使用 Skill 的结果，是用户自己的 AI 工作台账。

### 用户可以做什么

- 查看历史运行的 Skill
- 查看使用过的模型
- 查看输入和输出
- 查看运行成本和耗时
- 查看是否成功
- 查看本月省钱金额
- 换模型一键重跑同一次任务

### 适合什么场景

用户想复查之前的输出结果，或想用另一个模型重新跑同一个任务时，可以使用运行记录。

---

## 13. 模型 Key / BYOK 设置

### 这个模块是做什么的

BYOK 是 Bring Your Own Key，表示用户可以绑定自己的模型 API Key，用自己的模型额度运行 Skill。

### 用户可以做什么

- 设置自己的模型 API Key
- 清除已绑定的 Key
- 使用自己的模型额度运行 Skill
- 在平台代付和自有 Key 之间做选择

### 适合什么场景

用户已经有自己的模型 API 额度，或者希望自己控制模型来源和成本时，可以使用 BYOK。

---

## 14. 收藏与订阅

### 这个模块是做什么的

收藏用于保存用户感兴趣或常用的 Skill，并在 Skill 更新时获得提醒。

### 用户可以做什么

- 收藏常用 Skill
- 在控制台查看收藏列表
- 快速回到常用 Skill
- 接收收藏 Skill 的更新通知

### 适合什么场景

用户经常使用某个写作、总结、分析或格式化 Skill 时，可以收藏后长期使用。

---

## 15. 评论与评分

### 这个模块是做什么的

评论与评分用于记录真实使用体验，帮助其他用户判断 Skill 是否好用。

### 用户可以做什么

- 给 Skill 打分
- 写使用评价
- 提交失败案例
- 反馈兼容问题
- 帮助创作者改进 Skill

### 适合什么场景

用户运行 Skill 后，想分享体验、指出问题或帮助其他用户避坑时，可以留下评论。

---

## 16. 求术悬赏

### 这个模块是做什么的

求术悬赏用于把用户需求转化为新的 Skill 供给。用户可以发布自己想要的 Skill，并用术值悬赏创作者完成。

### 用户可以做什么

- 发布一个想要的 Skill 需求
- 设置术值悬赏
- 查看悬赏状态
- 创作者可以认领悬赏
- 创作者提交对应 Skill
- 发布人验收后发放术值
- 发布人取消未完成悬赏

### 适合什么场景

用户找不到合适 Skill，但能清楚描述自己的任务需求时，可以发布求术悬赏。

---

## 17. 创作者发布

### 这个模块是做什么的

创作者发布模块用于创建和维护自己的 Skill。

### 创作者可以做什么

- 创建新的 Skill
- 填写标题、简介、分类
- 配置输入字段
- 编写 Prompt
- 设置输出格式
- 设置推荐模型
- 发布版本
- 查看用户评论和运行反馈
- 更新维护自己的作品

### 适合什么场景

适合会写 Prompt、AI 工作流或任务模板，希望把能力分享给其他用户的人。

---

## 18. 术值

### 这个模块是做什么的

术值是平台内的贡献值，用来记录用户对平台内容和数据的贡献。

### 用户可以如何获得术值

- 发布优质 Skill
- 提交有效兼容报告
- 完成求术悬赏
- 贡献有效评论或反馈
- 参与平台认可的贡献行为

### 用户可以用术值做什么

- 发布求术悬赏
- 兑换部分 credit
- 体现账号贡献度

### 注意事项

术值不是现金，不能提现，也不用于随机抽奖。

---

## 19. credit

### 这个模块是做什么的

credit 是平台内用于模型调用的额度。当用户使用平台代付模型运行 Skill 时，会消耗 credit。

### 用户可以做什么

- 查看 credit 余额
- 查看运行消耗
- 通过平台规则获得或兑换 credit
- 使用 credit 运行 Skill

### 适合什么场景

用户没有自己的模型 API Key，或希望直接使用平台提供的模型调用能力时，会用到 credit。

---

## 20. 通知中心

### 这个模块是做什么的

通知中心用于提醒用户和自己相关的事件。

### 用户可以收到什么通知

- 收藏的 Skill 更新
- 已安装的 Skill 有新版本
- 求术悬赏状态变化
- 评论、审核、系统消息
- 账号或运行相关提醒

### 适合什么场景

用户希望及时知道自己关注的 Skill、悬赏或账号状态变化时，可以查看通知中心。

---

## 21. 邀请码

### 这个模块是做什么的

邀请码用于邀请新用户加入平台，并记录邀请关系。

### 用户可以做什么

- 查看自己的邀请码
- 复制邀请链接
- 邀请其他用户注册
- 查看邀请相关记录

### 适合什么场景

平台采用邀请或半邀请机制时，老用户可以通过邀请码邀请新用户加入。

---

## 22. 公开验签

### 这个模块是做什么的

公开验签用于展示 LocalScore 等分数快照的校验结果，帮助外部用户理解平台分数是否被无痕篡改。

### 用户可以做什么

- 查看分数快照数量
- 查看签名是否有效
- 查看快照哈希
- 查看签名时间
- 了解平台如何保存可复核的分数记录

### 适合什么场景

用户或合作方关心榜单可信度时，可以查看公开验签页面。

---

## 23. 开发者文档

### 这个模块是做什么的

开发者文档说明 Skill 如何作为 API 被调用，以及平台和模型网关如何配合。

### 用户可以做什么

- 查看 Skill 调用接口示例
- 了解运行接口的输入和返回
- 了解模型网关配置方式
- 进入管理后台入口

### 适合什么场景

开发者希望把衡术 Skill 接入自己的工具或自动化流程时，可以查看开发者文档。

---

## 24. 管理后台

### 这个模块是做什么的

管理后台是管理员、审核员和运营人员使用的模块，普通用户通常不需要进入。

### 管理员可以做什么

- 管理用户
- 管理 Skill、版本、制品和分类
- 查看运行记录和兼容报告
- 管理悬赏
- 管理邀请码
- 管理术值规则和流水
- 管理收藏、安装和 Runner 实例
- 审核评论和举报
- 管理媒体、站点设置和部署设置

### 适合什么场景

平台运营、内容审核、用户治理、系统配置和数据排查都在管理后台完成。

---

## 25. 部署设置

### 这个模块是做什么的

部署设置是管理员配置平台运行参数的地方。

### 管理员可以配置什么

- 站点地址
- 模型网关
- New API
- manifest 签名
- 备份确认
- 生产运行相关开关

### 适合什么场景

平台从本地测试切换到 NAS、服务器或公网生产环境时，需要管理员检查和填写部署设置。

---

## 26. 用户最常见的使用路径

### 26.1 普通用户

首页 -> Skill 市场 -> Skill 详情页 -> 在线试用 -> 收藏或运行 -> 控制台查看运行记录

### 26.2 本地 Runner 用户

安装 Runner -> 设备授权登录 -> 安装 Skill -> 本地运行 -> 回传兼容报告 -> 控制台查看 Runner 和运行记录

### 26.3 创作者

控制台 -> 发布 Skill -> 填写 Skill 信息 -> 发布版本 -> 查看评论和运行反馈 -> 持续更新

### 26.4 求术用户

悬赏区 -> 发布需求 -> 设置术值 -> 等待创作者提交 -> 验收 Skill -> 发放术值

### 26.5 管理员

后台登录 -> 检查部署设置 -> 管理 Skill 和用户 -> 处理举报 -> 查看运行与兼容数据
